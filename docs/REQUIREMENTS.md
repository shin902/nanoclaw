# NanoClaw 要件定義

プロジェクト作成者によるオリジナルの要件と設計決定事項。

---

## なぜこれが存在するのか

これは、OpenClaw (旧 ClawBot) に対する軽量で安全な代替手段です。OpenClaw プロジェクトは巨大化しすぎてしまいました。異なるゲートウェイを実行する 4〜5 個のプロセス、終わりのない設定ファイル、際限のない連携機能。エージェントが隔離されたプロセスで実行されないためセキュリティ上の悪夢となっており、アクセスすべきでないシステムの一部にアクセスするのを防ぐために、あらゆる種類の不完全な回避策が講じられています。コードベース全体を現実的に理解することは誰にとっても不可能です。実行するときは、ある種「出たとこ勝負」の状態です。

NanoClaw は、そのような混乱なしにコア機能を提供します。

---

## 哲学

### 理解できるほど小さいこと

コードベース全体が、読んで理解できるものであるべきです。1 つの Node.js プロセス。一握りのソースファイル。マイクロサービスなし、メッセージキューなし、抽象化レイヤーなし。

### 真の隔離によるセキュリティ

エージェントがリソースにアクセスするのを防ぐためにアプリケーションレベルの権限システムを使用するのではなく、エージェントを実際の Linux コンテナ内で実行します。隔離は OS レベルで行われます。エージェントは明示的にマウントされたものしか見ることができません。Bash アクセスが安全なのは、コマンドが Mac 上ではなくコンテナ内で実行されるためです。

### 一人のユーザーのために構築

これはフレームワークやプラットフォームではありません。私の特定のニーズを満たすための、動くソフトウェアです。私は WhatsApp と Email を使うので、WhatsApp と Email をサポートしています。Telegram は使わないので、Telegram はサポートしていません。あらゆる連携ではなく、私が実際に欲しい連携だけを追加します。

### カスタマイズ ＝ コード変更

設定ファイルの乱立はさせません。異なる動作をさせたい場合は、コードを修正してください。コードベースは十分に小さいため、これは安全かつ現実的です。トリガーワードのような極めて最小限のものだけが設定に含まれます。それ以外は、自分のやりたいようにコードを変えるだけです。

### AI ネイティブ開発

インストールウィザードは不要です — Claude Code がセットアップを案内します。監視ダッシュボードも不要です — 何が起きているか Claude Code に聞きます。精巧なログ UI も不要です — Claude にログを読ませます。デバッグツールも不要です — 問題を説明すれば Claude が直してくれます。

コードベースは、AI コラボレーターがいることを前提としています。Claude が常にそこにいるため、過度に自己文書化したり、自己デバッグ機能を備えたりする必要はありません。

### 機能よりもスキル

貢献者は「WhatsApp と並んで Telegram もサポート」を追加すべきではありません。代わりに、コードベースを変換する `/add-telegram` のようなスキルを提供すべきです。ユーザーはリポジトリをフォークし、スキルを実行してカスタマイズすることで、全員のユースケースを同時にサポートしようとする肥大化したシステムではなく、自分のニーズに正確に合致したクリーンなコードを手にすることになります。

## RFS (スキル募集)

貢献者に構築してほしいスキル：

### 通信チャネル
異なるメッセージングプラットフォームを追加または切り替えるためのスキル：
- `/add-telegram` - 入力チャネルとして Telegram を追加
- `/add-slack` - 入力チャネルとして Slack を追加
- `/add-discord` - 入力チャネルとして Discord を追加
- `/add-sms` - Twilio などによる SMS 連携を追加
- `/convert-to-telegram` - WhatsApp を完全に Telegram に置き換える

### コンテナランタイム
プロジェクトはデフォルトで Docker (クロスプラットフォーム) を使用しています。Apple Container を好む macOS ユーザー向け：
- `/convert-to-apple-container` - Docker から Apple Container に切り替える (macOS 専用)

### プラットフォーム対応
- `/setup-linux` - Linux 上でフルセットアップを動作させる (Docker への変換に依存)
- `/setup-windows` - WSL2 + Docker による Windows 対応

## ビジョン

WhatsApp 経由でアクセス可能な、最小限のカスタムコードで構築された個人用 Claude アシスタント。

**コアコンポーネント：**
- **Claude Agent SDK**: コアエージェントとして使用
- **コンテナ**: 隔離されたエージェント実行環境 (Linux VM)
- **WhatsApp**: 主要な I/O チャネル
- **永続メモリ**: 会話ごとおよびグローバルなメモリ
- **定期実行タスク**: Claude を実行し、メッセージを返信できる機能
- **ウェブアクセス**: 検索およびブラウジング機能
- **ブラウザ自動化**: agent-browser による自動化

**実装アプローチ：**
- 既存のツールを使用 (WhatsApp コネクタ、Claude Agent SDK, MCP サーバー)
- 最小限の接着コード
- 可能な限りファイルベースのシステム (メモリ用の CLAUDE.md、グループ用のフォルダ)

## アーキテクチャ決定事項

### メッセージルーティング
- ルーターが WhatsApp を監視し、設定に基づいてメッセージをルーティング
- 登録されたグループからのメッセージのみを処理
- トリガー： `@Andy` プレフィックス（大文字小文字を区別しない）、`ASSISTANT_NAME` 環境変数で設定可能
- 未登録のグループは完全に無視

### メモリシステム
- **グループごとのメモリ**: 各グループは独自の `CLAUDE.md` を持つフォルダを持つ
- **グローバルメモリ**: ルートの `CLAUDE.md` はすべてのグループから読み取れるが、書き込みは「メイン」（自分自身とのチャット）からのみ可能
- **ファイル**: グループは自身のフォルダ内でファイルを作成/読み取り、それらを参照できる
- エージェントはグループのフォルダ内で実行され、両方の `CLAUDE.md` ファイルを自動的に継承する

### セッション管理
- 各グループは会話セッションを維持 (Claude Agent SDK 経由)
- コンテキストが長くなりすぎるとセッションは自動的にコンパクション（要約）され、重要な情報を保持

### コンテナによる隔離
- すべてのエージェントはコンテナ（軽量な Linux VM）内で実行される
- エージェントの呼び出しごとに、ディレクトリをマウントしたコンテナが起動される
- コンテナはファイルシステムの隔離を提供し、エージェントはマウントされたパスしか見ることができない
- Bash アクセスが安全なのは、コマンドがホスト上ではなくコンテナ内で実行されるため
- コンテナ内の Chromium を使用した agent-browser によるブラウザ自動化

### 定期実行タスク
- ユーザーはどのグループからでも、定期実行または単発のタスクをスケジュールするよう Claude に依頼できる
- タスクは、それを作成したグループのコンテキストでフルエージェントとして実行される
- タスクは Bash (コンテナ内で安全) を含むすべてのツールにアクセスできる
- タスクはオプションで `send_message` ツールを使用してグループにメッセージを送信したり、サイレントに完了したりできる
- タスクの実行は、所要時間と結果とともにデータベースに記録される
- スケジュールタイプ： cron 式、インターバル (ms)、または単発 (ISO タイムスタンプ)
- メインから： 任意のグループのタスクをスケジュールでき、すべてのタスクを表示/管理できる
- 他のグループから： そのグループのタスクのみを管理できる

### グループ管理
- 新しいグループはメインチャネルを介して明示的に追加される
- グループは SQLite に登録される（メインチャネルまたは IPC の `register_group` コマンド経由）
- 各グループは `groups/` 配下に専用のフォルダを持つ
- グループは `containerConfig` を介して追加のディレクトリをマウントできる

### メインチャネルの特権
- メインチャネルは管理者/制御グループ（通常は自分自身とのチャット）
- グローバルメモリ (`groups/CLAUDE.md`) に書き込み可能
- 任意のグループのタスクをスケジュール可能
- すべてのグループのタスクを表示・管理可能
- 任意のグループに対して追加のディレクトリマウントを設定可能

## 連携ポイント

### WhatsApp
- WhatsApp Web 接続に baileys ライブラリを使用
- メッセージは SQLite に保存され、ルーターによってポーリングされる
- セットアップ中に QR コード認証を行う

### スケジューラ
- ビルトインスケジューラがホスト上で動作し、タスク実行のためにコンテナを起動
- カスタムの `nanoclaw` MCP サーバー (コンテナ内) がスケジューリングツールを提供
- ツール: `schedule_task`, `list_tasks`, `pause_task`, `resume_task`, `cancel_task`, `send_message`
- タスクは実行履歴とともに SQLite に保存される
- スケジューラループが 1 分ごとに実行期限のタスクを確認
- タスクはコンテナ化されたグループコンテキストで Claude Agent SDK を実行

### ウェブアクセス
- 内蔵の WebSearch および WebFetch ツール
- 標準的な Claude Agent SDK の機能

### ブラウザ自動化
- コンテナ内の Chromium を備えた agent-browser CLI
- 要素参照 (@e1, @e2 など) を使用したスナップショットベースの操作
- スクリーンショット、PDF、ビデオ録画
- 認証状態の永続化

## セットアップ & カスタマイズ

### 哲学
- 最小限の設定ファイル
- Claude Code を介してセットアップとカスタマイズを行う
- ユーザーはリポジトリをクローンし、Claude Code を実行して設定を行う
- 各ユーザーが自身のニーズに正確に合致したカスタムセットアップを手に入れる

### スキル
- `/setup` - 依存関係のインストール、WhatsApp 認証、スケジューラ設定、サービスの開始
- `/customize` - 機能追加のための汎用スキル (Telegram のような新しいチャネル、新しい連携、動作変更など)
- `/update` - アップストリームの変更を取り込み、カスタマイズとマージし、マイグレーションを実行

### デプロイ
- launchd を介してローカルの Mac 上で動作
- 単一の Node.js プロセスがすべてを処理
